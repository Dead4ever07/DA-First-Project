Validates user input and computes the appropriate route.

This function checks if the given inputs exist in the graph. Then, it determines the appropriate route computation based on the provided parameters. If the mode is "driving" with no constraints, it computes the best and alternative driving routes. Otherwise, it calculates a restricted or a driving-walking route.

Parameters
gPointer to the graph.
originThe ID of the source vertex.
destThe ID of the destination vertex.
vertexA vector of node IDs that should be avoided.
edgesA vector of pairs representing edges to be avoided.
middleThe ID of a mandatory intermediate vertex (-1 if not required).
modeThe selected travel mode ("driving" or "driving-walking").
maxWalkTimeThe maximum allowed walking time for mixed routes.
Returns
A formatted string containing the computed route or an error message if the input is invalid.
Note
Time Complexity: O((V + E) log V). The function may call driveRoute(), driveRestrictedRoute(), or driveWalkingRoute(), each executing Dijkstra’s algorithm.
